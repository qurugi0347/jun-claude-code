## 리뷰 관점

`code-reviewer` agent를 호출하여 아래 관점에서 구현 코드를 검토하고 Critical/Warning으로 분류한다.

아래는 기본 리뷰 관점이다. **구현의 목적, 도메인, 기술 스택에 따라 추가로 필요한 관점이 있다면 자유롭게 추가하여 리뷰한다.**

### 1. 목적 부합성
- 구현이 context.md 사용자 요청을 충족하는가?
- plan.md 설계 결정대로 구현되었는가?
- plan에 있으나 누락된 기능이 있는가?

### 2. 오버엔지니어링 검토
- 요구사항 대비 코드 복잡도가 적절한가?
- 사용하지 않을 추상화/인터페이스가 있는가?
- 더 단순한 구현 방법이 있는가?

### 3. 성능 관점
- N+1 쿼리, 불필요한 루프 등 성능 이슈가 있는가?
- 대량 데이터 처리 시 문제가 될 수 있는 부분이 있는가?
- 캐싱 적용 가능한 부분이 있는가?

### 4. 사용자 Flow 관점
- 에러 상태, 로딩 상태, 빈 상태가 처리되었는가?
- 사용자에게 적절한 피드백이 제공되는가?
- 예외 케이스가 처리되었는가?

### 5. 유지보수 관점
- 코드가 기존 프로젝트 패턴과 일관성이 있는가?
- 단일 책임 원칙이 지켜졌는가?
- 테스트 가능한 구조인가?

### 6. 보안 관점
- 사용자 입력이 적절히 검증되는가?
- 인증/권한 처리가 올바른가?
- 민감 정보가 노출되지 않는가?

## 출력 형식

```
# 코드 리뷰 결과

## 요약
- Critical: X건 / Warning: Y건
- 목적 달성도: (달성/미달성 기능 목록)
- 적용된 리뷰 관점: (기본 6개 + 추가된 관점 목록)

## Critical 이슈 (즉시 수정 필요)
| # | 파일:라인 | 관점 | 내용 | 수정 제안 |
|---|---------|------|------|---------|

## Warning (개선 권장)
| # | 파일:라인 | 관점 | 내용 | 수정 제안 |
|---|---------|------|------|---------|

## 긍정 평가
- 잘 구현된 부분 목록

## 미구현 항목
- plan.md에 있으나 구현되지 않은 기능 목록
```
